## 1. 引言

### 1.1 目的

本文档旨在详细定义FMSY文件转换应用的功能需求、行为规格及边界条件，作为后续系统设计和开发的依据。

### 1.2 范围

FMSY是一个数据库与FTP之间的数据文件转换应用，支持：
- 多种文件格式（DBF/XML/CSV/TXT）的双向传输
- 配置驱动的传输规则管理
- 多节点并发/串行任务处理
- 完整的稽核和错误处理机制

### 1.3 定义

| 术语 | 定义 |
|------|------|
| 上传（Upload） | 将FTP文件内容写入数据库表 |
| 下发（Download） | 将数据库表数据生成FTP文件 |
| 传输配置 | 定义一条传输任务的所有规则参数，存储在传输配置表中 |
| 传输场景 | 区分不同的上下传场景类型，包括UPLOAD_SINGLE/UPLOAD_MULTI/DOWNLOAD_SINGLE/DOWNLOAD_SINGLE_NODE/DOWNLOAD_MULTI_NODE |
| 指令表 | 触发传输任务的任务队列，存储待执行的指令 |
| 明细表 | 存储批量任务的子任务详情，用于指定文件上传下发或分桶下发 |
| 结果表 | 存储每个指令对应的传输结果 |
| S类型指令 | 分桶下发时创建的子指令，用于触发多节点并行处理分桶 |
| 主指令 | 创建S类型子指令的原始指令，可能为串行指令或批量指令 |
| 忽略字段 | 上传/下发时跳过的数据库字段，由忽略字段配置指定 |
| 替换字段 | 文件路径中配置的占位符字段名，需与`拆分字段配置`一致，取值从数据源获取 |
| 前置文件操作 | 任务执行前的文件检查操作，如检查就绪文件、标志文件 |
| 后置文件操作 | 任务执行后的文件处理操作，如生成标志文件、删除文件 |
| 串行指令 | 指令类型为空，相同类别代号和控制代号的多条就绪指令在多节点之间只能串行处理 |
| 批量指令 | 指令类型为'R'，需要根据指令ID去明细表查询批量上传/下传的指定文件名或分桶取值 |
| 协同指令 | 指令类型为'S'，需要多节点协同处理分桶下发的大数据量传输任务 |

### 2.2 核心流程

```
1. [启动] 系统启动时加载传输配置到内存，初始化转换器插件
2. [轮询] 轮询指令表，获取待执行指令（默认获取20条）
3. [竞争] 逐条竞争指令执行权，成功则启动线程异步处理
4. [前置处理] 执行前置文件操作（就绪文件检查、标志文件校验）
5. [执行] 根据传输场景执行文件转换：
   - UPLOAD_*：FTP文件 → 数据库
   - DOWNLOAD_*：数据库 → FTP文件
   - DOWNLOAD_MULTI_NODE：主节点创建S类型子指令分发任务，各节点协同执行
6. [后置处理] 执行后置文件操作（标志文件生成、反馈文件等）
7. [记录] 更新指令状态，写入结果表
8. [返回] 继续轮询下一批指令
```

**说明**：
- 步骤4-6构成完整的单次传输任务处理流程
- DOWNLOAD_MULTI_NODE场景下，步骤5执行过程中会先创建S类型子指令分发到各节点，各节点竞争处理分桶

## 3. 状态值规范

所有涉及处理状态的结果取值统一为以下五种：

| 取值 | 名称 | 说明 |
|------|------|------|
| 空 | 就绪 | 插入后的初始状态 |
| P | 处理中 | 竞争后被处理中 |
| Y | 成功 | 完全成功，文件已生成或数据表已落库 |
| N | 跳过 | 下发时空记录且不允许发空文件，或上传时标志文件未到达 |
| E | 异常 | 转换过程发生异常导致未完成，或落库后回滚等失败 |

### 3.1 各表状态字段说明

**指令表.处理状态**：
| 状态 | 说明 |
|------|------|
| 空 | 就绪，等待被执行 |
| P | 处理中，正在执行 |
| Y | 完成，完全成功 |
| N | 跳过，标志文件未到达或空记录不允许发空文件 |
| E | 失败，发生异常 |

**明细表.处理状态**：
| 状态 | 说明 |
|------|------|
| 空 | 就绪，等待被执行（分桶未分配） |
| P | 处理中，正在生成分桶文件 |
| Y | 成功，分桶文件已生成 |
| N | 跳过，分桶数据为空且不允许发空文件 |
| E | 异常，分桶文件生成失败或回滚 |

**结果表.处理结果**：
| 状态 | 说明 |
|------|------|
| Y | 成功 |
| N | 跳过，标志文件未到达或空记录不允许发空文件 |
| E | 异常，转换过程发生异常或回滚失败 |
| P | 处理中 |

**注意**：结果表在指令完成后才写入，不存在空状态的未处理结果。

## 4. 占位符说明

### 4.1 占位符语法

使用 `$变量名$` 格式，如 `$YYYYMMDD$`、`$FIELD$`。

### 4.2 占位符类型

| 类型 | 示例 | 取值来源 | 适用场景 |
|------|------|----------|----------|
| 日期 | `$YYYYMMDD$` | 系统当前日期 | 上传/下发 |
| 时间 | `$YYYYMMDDHHmmss$` | 系统当前时间 | 上传/下发 |
| 指令字段 | `$EXTRA_INFO$` | 指令表额外信息字段 | 上传/下发 |
| 替换字段 | `$REGION$`、`$STATUS$`、`$FILE_NAME$` | 按`拆分字段配置`顺序从数据源取值 | 所有场景 |

**说明**：
- 替换字段占位符（如`$REGION$`、`$STATUS$`）的字段名必须与`拆分字段配置`中配置的字段名完全一致
- 多替换字段时，文件路径中可配置多个占位符，如`$REGION$_$STATUS$.dbf`
- 替换字段取值按`拆分字段配置`中的字段顺序从数据源获取：
  - 上传场景（UPLOAD_*）：从FTP文件名或明细表指定字段取值获取
  - 下发场景（DOWNLOAD_*）：从数据库表字段或明细表指定字段取值获取
- 日期/时间占位符在运行时解析，支持格式化如`$YYYYMMDDHHmmss$`

### 4.3 文件命名示例

---

#### 单文件上传（UPLOAD_SINGLE）

- 文件路径：`/data/upload/student_$YYYYMMDD$.csv`
- 匹配文件：`student_20260525.csv`
- 标志文件：`student_20260525.csv.OK`（如配置了CHECK_FLAG）

---

#### 目录多文件上传（UPLOAD_MULTI，指令类型='空'）

- 文件路径：`/data/upload/$YYYYMMDD$/*.csv`
- 匹配目录：`/data/upload/20260525/`
- 匹配文件：`file1.csv`、`file2.csv`、`file3.csv`
- 标志文件：`/data/upload/20260525/*.OK`（如配置了CHECK_READY）

---

#### 指定多文件上传（UPLOAD_MULTI，指令类型='R'）

- 拆分字段配置：`FILE_NAME`（仅用于文件名替换）
- 明细表记录1：指定字段取值=`student`
- 明细表记录2：指定字段取值=`teacher`
- 文件路径：`/data/upload/$FILE_NAME$.csv`
- 生成文件：`student.csv`、`teacher.csv`

---

#### 单文件下传（DOWNLOAD_SINGLE）

- 文件路径：`/data/export/student.dbf`
- 生成文件：`student.dbf`
- 标志文件：`student.dbf.OK`（如配置了后置操作）

---

#### 多文件下传（DOWNLOAD_SINGLE_NODE，指令类型='空'）

- 拆分字段配置：`REGION`
- 文件路径：`/data/export/$YYYYMMDD$/$REGION$.dbf`
- 目录名：`/data/export/20260525/`
- 生成文件：`20260525EAST.dbf`、`20260525WEST.dbf`、`20260525NORTH.dbf`
- 子文件标志：`20260525EAST.dbf.ready`（如配置了GENERATE_SUB_FLAG）
- 总标志：`/data/export/20260525.flg`（如配置了GENERATE_TOTAL_FLAG）

---

#### 指定拆分下传（DOWNLOAD_SINGLE_NODE，指令类型='R'）

- 拆分字段配置：`REGION,STATUS`
- 明细表记录1：指定字段取值=`EAST,ACTIVE`
- 明细表记录2：指定字段取值=`WEST,NORMAL`
- 文件路径：`/data/export/$YYYYMMDD$/$REGION$_$STATUS$.dbf`
- 目录名：`/data/export/20260525/`
- 生成文件：`20260525EAST_ACTIVE.dbf`、`20260525WEST_NORMAL.dbf`
- 子文件标志：`20260525EAST_ACTIVE.dbf.ready`（如配置了GENERATE_SUB_FLAG）
- 总标志：`/data/export/20260525.flg`（如配置了GENERATE_TOTAL_FLAG）

---

#### 多节点协调拆分下传（DOWNLOAD_MULTI_NODE）

与「多文件下传」或「指定拆分下传」的文件命名方式相同，区别在于：
- 主节点创建S类型子指令，各节点竞争处理不同分桶
- 最终汇总生成总标志文件

---

**汇总：占位符与场景对照表**

| 场景 | 拆分字段配置示例 | 文件路径示例 | 占位符使用 |
|------|----------------|-------------|------------|
| 单文件上传 | - | `/data/upload/student_$YYYYMMDD$.csv` | 日期占位符 |
| 目录多文件上传 | - | `/data/upload/$YYYYMMDD$/*.csv` | 日期占位符、通配符 |
| 指定多文件上传 | `FILE_NAME` | `/data/upload/$FILE_NAME$.csv` | 替换字段占位符 |
| 单文件下传 | - | `/data/export/student.dbf` | 无占位符 |
| 多文件下传 | `REGION` | `/data/export/$YYYYMMDD$/$REGION$.dbf` | 日期+替换字段占位符 |
| 指定拆分下传 | `REGION,STATUS` | `/data/export/$YYYYMMDD$/$REGION$_$STATUS$.dbf` | 日期+多替换字段占位符 |

## 5. 指令类型说明

| 指令类型取值 | 名称 | 说明 |
|-------------|------|------|
| 空 | 串行指令 | 相同类别代号和控制代号的多条就绪指令在多节点之间只能串行处理，这是大多数指令的类别 |
| 'R' | 批量指令 | 需要根据指令ID去明细表查询批量上传/下传的指定文件名或分桶取值，实现指定文件的上下传 |
| 'S' | 协同指令 | 需要多节点协同拆分文件的大数据量传输任务，此时对应的主指令的类型可能为空也可能为'R'，区别在于主指令是写明细还是更新明细 |

## 5.5 传输流程说明

传输流程分为三大块：**前置处理 → 文件传输 → 后置处理**

### 5.5.1 前置处理（处理每个文件前）

**配置字段：前置文件操作**
- 格式：`操作类型;参数1;参数2,操作类型;参数1;参数2,...`
- 各操作按配置顺序依次执行

| 操作类型 | 参数 | 说明 |
|----------|------|------|
| CHECK_READY | 文件名模式 | 检查文件是否存在，不存在则跳过（结果为'N'），结束 |
| CHECK_FLAG | 文件名模式;内容模式 | 检查标志文件是否存在，不存在则跳过；存在则读取内容（期望值），若匹配"空"模式则跳过校验，否则用数据文件内容计算实际值与期望值比较 |

**前置处理事件流**：
1. 解析前置文件操作配置，获取操作列表
2. 按配置顺序依次执行每个操作：
   a. **CHECK_READY**：
      - 检查文件是否存在，不存在则跳过（结果为'N'），结束前置处理
   b. **CHECK_FLAG**：
      - 检查文件是否存在，不存在则跳过（结果为'N'），结束前置处理
      - 读取标志文件内容（期望值，如记录数）
      - 若内容匹配"空"模式（如NULL/EMPTY/0），跳过校验，结束
      - 若内容为非空，用数据文件内容计算实际校验值，与标志文件内容（期望值）比较
        - 一致：校验通过，结束
        - 不一致：标记异常（结果为'E'），结束前置处理

**说明**：内容模式用于判断标志文件是否表示"无数据"。如果标志文件内容为空或表示无数据，则跳过标志文件校验，改用数据文件本身进行校验计算。

### 5.5.2 文件传输

执行文件解析、字段映射、数据落库或文件生成。

### 5.5.3 后置处理（处理每个文件后）

**配置字段：后置文件操作**
- 格式：`操作类型;参数1;参数2,操作类型;参数1;参数2,...`
- 各操作按配置顺序依次执行

| 操作类型 | 参数 | 说明 |
|----------|------|------|
| GENERATE_FEEDBACK | 文件名模式;内容取值 | 生成反馈文件，文件名模式指定文件名，内容取值指定文件内容 |
| RENAME | 原文件模式;目标文件模式 | 重命名文件 |
| DELETE | 文件名模式 | 删除匹配的文件 |
| SEND_MESSAGE | 消息内容;目标配置 | 发送消息（具体实现依赖配置） |
| GENERATE_SUB_FLAG | 文件名模式;内容取值 | 生成子文件标志文件（分桶下发场景） |
| GENERATE_TOTAL_FLAG | 文件名模式;内容取值 | 生成总标志文件（分桶下发场景） |

**分桶下发场景下的标志文件生成控制**：

| 标志类型 | 默认行为 | 可配置项 |
|----------|----------|----------|
| 子文件标志文件 | 每个子文件生成后自动生成 `{subfile}.ok` | 生成开关、文件名模式、内容取值 |
| 总标志文件 | 所有子文件完成后根据状态生成 `{dirname}.flg` | 生成开关、文件名模式、内容取值 |

**文件名模式中的变量**：
- `{subfile}` - 替换为当前子文件名（不含扩展名）
- `{dirname}` - 替换为当前目录名
- 内容取值支持固定值或变量（如 `{subfile}_OK`）

**后置处理事件流**：
1. 解析后置文件操作配置，获取操作列表
2. 按配置顺序依次执行每个操作：
   a. **GENERATE_FEEDBACK**：生成反馈文件
   b. **GENERATE_SUB_FLAG**：生成子文件标志（如配置了）
   c. **GENERATE_TOTAL_FLAG**：生成总标志文件（如配置了）
   d. **RENAME**：重命名匹配的文件
   e. **DELETE**：删除匹配的文件
   f. **SEND_MESSAGE**：发送消息

### 5.5.4 配置格式示例

```
前置文件操作：*.OK;NULL,*.FLG;NULL
  解释：检查*.OK是否存在，不存在则N；存在则读取内容，若为NULL则用数据文件校验；再检查*.FLG

前置文件操作：*.OK,*.FLG;0
  解释：检查*.OK是否存在；检查*.FLG，若内容为0则用数据文件校验，否则用FLG内容校验

后置文件操作：DELETE;*.OK,GENERATE_FEEDBACK;*.RET;SUCCESS,RENAME;*.FLG;*.DONE
  解释：先删除*.OK文件，再生成*.RET反馈文件（内容为SUCCESS），最后将*.FLG重命名为*.DONE
```

## UC-01：轮询指令表

### 7.1.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-01 |
| 用例名称 | 轮询指令表 |
| 参与者 | FMSY系统（自动执行） |
| 触发条件 | 系统启动后持续运行 |
| 前置条件 | 系统已启动，数据库连接正常 |

### 7.1.2 事件流

**基本事件流**：
1. 系统以配置的轮询间隔（默认10秒）定时执行轮询
2. 查询当前节点超时任务（执行时间超过1小时），自动释放：
   ```sql
   UPDATE 指令表
   SET 处理状态='E', 处理结束时间=NOW()
   WHERE 处理状态='P'
     AND 处理节点='当前节点ID'
     AND 处理起始时间 < NOW() - INTERVAL 1 HOUR
   ```
   - 对每个超时任务，插入结果表：`处理结果='E'，结果说明='执行超时自动释放'`
3. 执行SQL查询（获取就绪指令 + 获取正在处理的指令）：
   ```sql
   -- 获取正在处理的指令（加载到内存，供串行约束检查使用）
   SELECT 处理节点, 类别代号, 控制代号, 指令类型
   FROM 指令表
   WHERE 处理状态='P' AND 处理节点 IS NOT NULL;
   ```
   ```sql
   -- 获取就绪指令
   SELECT 自增列, 类别代号, 控制代号, 指令类型, 稽核数, 额外信息
   FROM 指令表
   WHERE 处理状态='空' AND 处理节点 IS NULL
   ORDER BY 自增列 ASC
   LIMIT 20
   ```
4. 将正在处理的指令加载到内存Map：
   - key: `类别代号+控制代号` → value: `{节点ID列表, 是否有S类型}`
   - 供后续串行约束检查使用
5. 如果有查询结果，遍历这批指令
6. 对每条指令调用UC-17进行串行约束检查（内存检查）
7. 串行检查通过后，调用UC-02尝试竞争
8. 竞争成功 → 启动新线程异步处理，主线程继续下一条
9. 竞争失败 → 主线程继续下一条
10. 所有指令处理完后，等待轮询间隔后重新查询

**超时检测说明**：
- 每轮询检测一次当前节点超时任务（执行超1小时）
- 超时任务自动更新为'E'状态
- 对每个超时任务写入结果表：`处理结果='E'，结果说明='执行超时自动释放'`
- 防止节点异常时任务永久卡在P状态

**异常事件流**：
- E1：数据库连接失败 → 记录错误日志，等待重试
- E2：查询超时 → 记录警告日志，继续下次轮询

**优化说明**：
- 正在处理的指令信息每轮询一次更新一次
- 串行约束检查直接在内存中进行，无需逐条查询数据库

### 7.1.3 后置条件

- 获取一批待执行的指令，逐条竞争处理
- 系统继续轮询

### 7.1.4 业务规则

- 轮询间隔可配置，最小值5秒，最大值300秒
- 每批次获取指令数可配置，默认20
- 任务执行超时时间可配置，默认1小时，超时自动释放
- 查询结果按自增列升序排序，确保FIFO执行顺序
- 对于相同类别代号+控制代号的就绪指令只取最早那条（去重），保证每个节点竞争同一条指令
- 多节点同时轮询时，各自获取的列表自然不同，任务分散到各节点

## UC-02：竞争指令执行权

### 7.2.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-02 |
| 用例名称 | 竞争指令执行权 |
| 参与者 | FMSY系统节点 |
| 触发条件 | UC-17串行约束检查通过后 |
| 前置条件 | 存在状态为"空"且未分配节点的指令 |

### 7.2.2 事件流

**基本事件流**：
1. 节点执行竞争SQL：
   ```sql
   UPDATE 指令表
   SET 处理节点='节点ID', 处理起始时间=NOW(), 处理状态='P'
   WHERE 自增列=? AND 处理状态='空' AND 处理节点 IS NULL
   ```
2. 检查影响行数
3. 如果影响行数=1，表示竞争成功
4. 如果影响行数=0，表示竞争失败（已被其他节点抢走）
5. 返回竞争结果

**S类型指令竞争SQL**（与普通指令相同）：
```sql
UPDATE 指令表
SET 处理节点='节点ID', 处理起始时间=NOW(), 处理状态='P'
WHERE 自增列=? AND 处理状态='空' AND 处理节点 IS NULL
```

**说明**：
- S类型子指令与主指令由相同节点处理时，不受串行约束限制
- 主节点处理主指令期间可以同时竞争处理S类型子指令

**异常事件流**：
- E1：UPDATE执行失败 → 记录错误，返回竞争失败

### 7.2.3 后置条件

- 竞争成功：该节点获得该指令的执行权，启动新线程处理
- 竞争失败：该指令已被其他节点抢走，返回UC-01继续处理下一条

### 7.2.4 业务规则

- 节点ID格式：`hostname`，不包含时间戳，便于查看任务归属
- 使用事务确保竞争原子性
- 竞争成功则启动新线程异步处理，实现负载均衡
- 竞争时直接设置处理状态为'P'（处理中）

| 配置项 | 说明 |

| 覆盖标识 | Y/N |

## UC-05：执行上传任务

### 7.5.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-05 |
| 用例名称 | 执行上传任务 |
| 参与者 | FMSY系统 |
| 触发条件 | UC-04获取到上传类型配置（UPLOAD_SINGLE/UPLOAD_MULTI） |
| 前置条件 | 配置中传输场景以UPLOAD开头 |

### 7.5.2 事件流

**上传场景总览**：

| 传输场景配置值 | 场景 | 触发条件 | 说明 |
|---------------|------|----------|------|
| UPLOAD_SINGLE | 单文件上传 | 占位符匹配到1个文件 | 处理单个文件 |
| UPLOAD_MULTI | 目录多文件上传 | 传输场景=UPLOAD_MULTI，占位符匹配到多个文件 | 多线程并行处理 |
| UPLOAD_MULTI | 指定多文件上传 | 传输场景=UPLOAD_MULTI，指令类型='R' | 根据明细表指定文件上传 |

**场景匹配优先级**：
1. 首先根据配置的`传输场景`确定大类
2. UPLOAD_SINGLE：占位符匹配到1个文件时触发
3. UPLOAD_MULTI：根据`指令类型`决定处理分支
   - 指令类型='空'：目录多文件上传（匹配多个文件）
   - 指令类型='R'：指定多文件上传（根据明细表指定文件）

---

#### 7.5.2.1 单文件上传

**适用条件**：传输场景=UPLOAD_SINGLE

**事件流**：
1. 解析文件路径，识别是否有占位符
2. 如果有占位符，调用UC-10进行解析替换
3. 连接到FTP服务器
4. **执行前置处理**（就绪文件检查、标志文件校验等）
   - 标志文件未到达 → 文件结果为'N'，结束
   - 标志文件校验失败 → 文件结果为'E'，结束
5. 对单个文件执行：
   a. 调用UC-11执行前稽核
      - 稽核失败 → 调用UC-14处理错误文件，文件结果为'E'
   c. 根据解析器类型选择对应转换器
   d. 解析文件内容，按字段映射写入数据库
   e. 检查解析后的数据是否为空：
      - 空数据 + 空数据处理=ERROR → 不写入数据库，文件结果为'E'
      - 空数据 + 空数据处理=ALLOW → 不写入数据库，文件结果为'Y'
      - 非空数据 → 继续写入数据库
   f. 如果配置清表标识=Y，清空目标表后再写入
   g. 调用UC-12执行后稽核
      - 稽核失败 → 调用UC-14处理错误文件并回滚，文件结果为'E'
   h. **执行后置处理**（生成标志文件、反馈文件等）
   i. 文件结果为'Y'
6. 调用UC-19写入结果表
7. 更新指令状态为'Y'（成功）或'E'（失败）或'N'（跳过）

---

#### 7.5.2.2 目录多文件上传

**适用条件**：传输场景=UPLOAD_MULTI，指令类型='空'，占位符解析后匹配到多个文件

**事件流**：
1. 解析文件路径，识别是否有占位符
2. 如果有占位符，调用UC-10进行解析替换
3. 连接到FTP服务器
4. 匹配到N个文件，启动多线程并行处理（线程数=MIN(N, 配置并发数)）
5. **主线程执行前置处理**（就绪文件检查、标志文件校验等）
6. 对每个文件执行（各线程独立）：
   a. **执行前置处理**（就绪文件检查、标志文件校验等）
      - 标志文件未到达 → 文件结果为'N'，跳过
      - 标志文件校验失败 → 文件结果为'E'
   b. 调用UC-11执行前稽核
      - 稽核失败 → 调用UC-14处理错误文件，文件结果为'E'
   c. 根据解析器类型选择对应转换器
   d. 解析文件内容，按字段映射写入数据库
   e. 检查解析后的数据是否为空：
      - 空数据 + 空数据处理=ERROR → 不写入数据库，文件结果为'E'
      - 空数据 + 空数据处理=ALLOW → 不写入数据库，文件结果为'Y'
      - 非空数据 → 继续写入数据库
   f. 如果配置清表标识=Y，清空目标表后再写入
   g. 调用UC-12执行后稽核
      - 稽核失败 → 调用UC-14处理错误文件并回滚，文件结果为'E'
   h. **执行后置处理**（生成标志文件、反馈文件等）
   i. 文件结果为'Y'
7. 主线程等待所有子任务完成
8. 调用UC-19写入结果表
9. 更新指令状态：
   - 所有文件成功 → 'Y'
   - 部分成功 → 'Y'（部分成功视为成功）
   - 全部跳过 → 'N'
   - 全部失败 → 'E'

**并行控制**：
- 单节点多线程并发数：由传输配置中的`并发数`字段控制，默认2-3，避免事务爆炸
- 多节点并发数：由配置控制（最大并发节点数）
- 每个文件独立事务，互不影响
- 数据库写入由数据库事务控制，无需节点加锁

---

#### 7.5.2.3 指定多文件上传

**适用条件**：传输场景=UPLOAD_MULTI，指令类型='R'，明细表有指定文件记录

**事件流**：
1. 根据指令ID查询明细表，获取指定文件列表
2. 遍历明细表记录，对每条记录：
   a. 根据明细表的指定字段名称和指定字段取值填充占位符生成文件路径
   b. 连接到FTP服务器（如尚未连接）
   c. **执行前置处理**（就绪文件检查、标志文件校验等）
   d. 调用UC-11执行前稽核（使用明细表的稽核数）
      - 稽核失败 → 调用UC-14处理错误文件，文件结果为'E'
   e. 根据解析器类型选择对应转换器
   f. 解析文件内容，按字段映射写入数据库
   g. 检查解析后的数据是否为空：
      - 空数据 + 空数据处理=ERROR → 不写入数据库，文件结果为'E'
      - 空数据 + 空数据处理=ALLOW → 不写入数据库，文件结果为'Y'
      - 非空数据 → 继续写入数据库
   h. 如果配置清表标识=Y，清空目标表后再写入
   i. 调用UC-12执行后稽核
      - 稽核失败 → 调用UC-14处理错误文件并回滚，文件结果为'E'
   j. **执行后置处理**（生成标志文件、反馈文件等）
   k. 更新明细表的处理状态（Y/E/N）
3. 调用UC-19写入结果表
4. 更新指令状态：
   - 所有明细成功 → 'Y'
   - 部分成功 → 'Y'
   - 全部跳过 → 'N'
   - 全部失败 → 'E'

**明细表字段使用**：
| 字段 | 用途 |
|------|------|
| 指定文件名 | 填充占位符用于定位FTP上的文件 |
| 指定字段名称 | 额外落库字段名称（多字段逗号分割） |
| 指定字段取值 | 对应字段的值（与指定字段名称顺序对应） |
| 稽核数 | 该文件的记录数，用于前稽核 |

### 7.5.3 错误处理策略

| 场景 | 处理策略 |
|------|----------|
| 单文件失败 | 直接回滚整个任务，指令状态为'E' |
| 多文件部分失败 | 只回滚失败文件，正常数据提交；指令结果仍为'Y' |
| 有指定文件失败 | 明细表对应状态更新为'E' |
| 文件跳过（标志文件未到达） | 文件结果为'N'，继续下一文件 |
| 空数据不允许为空 | 文件结果为'E' |
| 空数据允许为空 | 文件结果为'Y'，不写入数据库 |

### 7.5.4 空数据处理

上传时，当解析后的数据为空（记录数为0）：
- ERROR：不允许为空，文件结果为'E'，不写入数据库
- ALLOW：允许为空，文件结果为'Y'，不写入数据库

**说明**：上传结果为'N'代表文件未就绪（标志文件未到达），与空数据处理无关。

## UC-06：执行下发任务

### 7.6.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-06 |
| 用例名称 | 执行下发任务 |
| 参与者 | FMSY系统 |
| 触发条件 | UC-04获取到下发类型配置（DOWNLOAD_*） |
| 前置条件 | 配置中传输场景以DOWNLOAD开头 |

### 7.6.2 事件流

**下发场景总览**：

| 传输场景配置值 | 场景 | 触发条件 | 说明 |
|---------------|------|----------|------|
| DOWNLOAD_SINGLE | 单文件下传 | 文件路径是具体文件，无拆分字段配置 | 不创建目录，直接生成文件 |
| DOWNLOAD_SINGLE_NODE | 单节点多文件下传 | 指令类型='R'，明细表有指定文件记录 | 根据明细表指定生成多个文件，单节点处理 |
| DOWNLOAD_SINGLE_NODE | 单节点多文件下传（全表） | 指令类型='空'，配置拆分字段 | 单节点直接遍历分桶生成文件，不写明细表 |
| DOWNLOAD_MULTI_NODE | 多节点协调拆分下传 | 配合S类型子指令 | 多节点协同处理分桶下发 |

**场景匹配优先级**：
1. 首先根据配置的`传输场景`确定大类
2. 然后根据`指令类型`（空/R）和`拆分字段配置`确定具体处理分支
3. `DOWNLOAD_MULTI_NODE`场景下，指令类型决定是否需要先拆分数据到明细表

---

#### 7.6.2.1 单文件下传

**适用条件**：传输场景=DOWNLOAD_SINGLE

**事件流**：
1. 解析文件名，识别占位符
2. 如果有占位符，调用UC-10解析替换
3. 不创建目录，直接生成文件到目标路径
4. 检查目标路径是否有同名文件：
   - 如果存在且覆盖标识=Y，或不存在 → 继续
   - 如果存在且覆盖标识=N，检查标志文件是否存在：
     - 存在则报错结束，结果为'E'
     - 不存在则覆盖
5. **执行前置处理**（就绪文件检查、标志文件校验等）
6. 调用UC-11执行前稽核
7. 根据解析器类型选择对应转换器
8. 从数据库读取数据
9. 应用忽略字段配置，排除指定字段
10. 检查数据是否为空：
    - 空数据 + 空数据处理=ERROR → 结果为'E'，任务失败
    - 空数据 + 空数据处理=SEND_EMPTY → 正常生成空文件，结果为'Y'
    - 空数据 + 空数据处理=SKIP → 不生成文件，结果为'N'
    - 非空数据 → 正常生成文件
11. 生成目标文件
12. **执行后置处理**（生成标志文件、反馈文件等）
13. 调用UC-12执行后稽核
14. 调用UC-19写入结果表
15. 更新指令状态为'Y'（成功）或'E'（失败）

#### 7.6.2.2 单节点多文件下传（指定）

**适用条件**：传输场景=DOWNLOAD_SINGLE_NODE，指令类型='R'

**事件流**：
1. 根据指令ID查询明细表，获取指定文件列表
2. 调用UC-18创建下发目录（目录名在文件路径里），目录不存在时创建
3. **执行前置处理**（就绪文件检查、标志文件校验等）
4. 遍历明细表，对每条记录：
   a. 根据明细表的指定字段名称和指定字段取值填充占位符确定输出文件
   b. （覆盖检查在UC-18执行总标志文件检查，不再逐文件检查）
   c. 调用UC-11执行前稽核（使用明细表的稽核数）
   d. 根据解析器类型选择对应转换器
   e. 从数据库读取数据，应用忽略字段配置
   f. 检查数据是否为空：
      - 空数据 + 空数据处理=ERROR → 该明细结果为'E'，继续下一明细
      - 空数据 + 空数据处理=SEND_EMPTY → 正常生成空文件，该明细结果为'Y'
      - 空数据 + 空数据处理=SKIP → 不生成文件，该明细结果为'N'
      - 非空数据 → 正常生成文件，该明细结果为'Y'
   g. 生成目标文件
   h. 根据后置处理中的`GENERATE_SUB_FLAG`配置生成子文件标志文件
   i. 更新明细表处理状态（Y/N/E）
5. 所有子文件完成后，根据`GENERATE_TOTAL_FLAG`配置和最终状态决定是否生成总标志文件
6. 调用UC-19写入结果表
7. 更新指令状态：
   - 所有明细成功 → 'Y'
   - 部分成功 → 'Y'
   - 全部跳过 → 'N'
   - 全部失败 → 'E'

#### 7.6.2.3 单节点多文件下传（全表）

**适用条件**：传输场景=DOWNLOAD_SINGLE_NODE，指令类型='空'，配置拆分字段

**说明**：单节点处理，不创建S类型子指令，不写明细表，直接遍历分桶生成文件

**事件流**：
1. 主节点竞争获得指令执行权
2. 查询数据库表，按拆分字段分组获取所有分桶取值
3. 调用UC-18创建下发目录（目录名在文件路径里），目录不存在时创建
4. **执行前置处理**（就绪文件检查、标志文件校验等）
5. 遍历每个分桶，对每个分桶执行：
   a. 根据分桶的拆分字段取值填充占位符确定输出文件名
   b. （覆盖检查在UC-18执行总标志文件检查，不再逐文件检查）
   c. 调用UC-11前稽核
   d. 根据解析器类型选择对应转换器
   e. 从数据库读取数据，应用忽略字段配置，按分桶字段取值筛选
   f. 检查数据是否为空：
      - 空数据 + 空数据处理=ERROR → 该分桶结果为'E'，继续下一分桶
      - 空数据 + 空数据处理=SEND_EMPTY → 正常生成空文件，该分桶结果为'Y'
      - 空数据 + 空数据处理=SKIP → 不生成文件，该分桶结果为'N'
      - 非空数据 → 正常生成文件，该分桶结果为'Y'
   g. 生成目标文件到下发目录
   h. 根据后置处理中的`GENERATE_SUB_FLAG`配置生成子文件标志文件
6. 所有子文件完成后，根据`GENERATE_TOTAL_FLAG`配置和最终状态决定是否生成总标志文件
7. 调用UC-19写入结果表
8. 更新指令状态：
   - 所有分桶成功 → 'Y'
   - 部分成功 → 'Y'
   - 全部跳过 → 'N'
   - 全部失败 → 'E'

**与单节点多文件下传（指定）的区别**：
- 指定多文件下传：明细表已有所需的分桶/文件数据，直接使用
- 全表多文件下传：单节点直接查询数据库表获取分桶取值，不写明细表

#### 7.6.2.4 多节点协调拆分下传

**适用条件**：传输场景=DOWNLOAD_MULTI_NODE

**初始化阶段（区分指令类型）**：

| 指令类型 | 主指令行为 |
|----------|-----------|
| '空'（明细表无分桶数据） | 1. 查询数据库表，按拆分字段分组获取所有分桶取值<br>2. 在明细表创建N条分桶记录（状态='空'）<br>3. 插入M条S类型子指令 |
| 'R'（明细表已有分桶数据） | 1. 查询明细表中该主指令ID对应的所有分桶<br>2. 对每个分桶更新稽核数为该分桶对应的实际记录数<br>3. 插入M条S类型子指令 |

**主节点/从节点行为**：
1. 节点通过UC-09轮询竞争S类型子指令
2. 竞争获得S类型子指令后：
   a. 轮询该主指令对应的就绪分桶（每次获取一批）
   b. 遍历这批分桶，逐条竞争处理：
      - 执行竞争UPDATE：`SET 处理节点=?, 处理状态='P' WHERE 自增列=? AND 处理状态='空'`
      - 竞争成功 → 处理该分桶
      - 竞争失败 → 继续下一条
   c. 对每个竞争到的分桶：
      - 调用UC-11前稽核（使用明细表的稽核数）
      - 根据分桶的指定字段取值筛选数据
      - 生成文件到下发目录
      - 根据`GENERATE_SUB_FLAG`配置生成子文件标志
      - 调用UC-12后稽核
      - 更新分桶处理状态为'Y'/'E'/'N'
   d. 该S类型子指令处理完成，更新状态为'Y'或'E'
   e. 调用UC-19写入该S类型子指令的结果记录

**分桶竞争机制**：
```sql
UPDATE 明细表
SET 处理节点='节点ID', 处理状态='P'
WHERE 自增列=? AND 处理状态='空'
```
- 影响行数=1：竞争成功，处理该分桶
- 影响行数=0：已被抢走，继续竞争下一个

**主指令与S类型子指令的区别**：

| 字段 | 主指令 | S类型子指令 |
|------|--------|-------------|
| 类别代号 | 与配置表对应 | 与主指令相同 |
| 控制代号 | 与配置表对应 | 与主指令相同 |
| 指令类型 | 空（无明细）或'R'（已有明细） | 'S' |
| 额外信息 | 可存储子指令数量 | 存储主指令ID，用于检索明细表 |
| 稽核数 | 由插入时写入 | -1（不稽核） |

主指令和子指令通过相同的类别代号和控制代号从配置表获取传输配置。

**S类型子指令创建SQL**：
```sql
INSERT INTO 指令表 (类别代号, 控制代号, 指令类型, 额外信息, 稽核数)
VALUES (主类别代号, 主控制代号, 'S', 主指令ID, -1)
```

### 7.6.3 异常事件流

- E1：数据库查询失败 → 记录错误，指令状态更新为'E'
- E2：前稽核失败 → 记录错误，指令状态更新为'E'
- E3：文件生成失败 → 记录错误，终止后续生成
- E4：后稽核失败 → 记录错误，指令状态更新为'E'
- E5：覆盖检查未通过 → 记录警告，跳过该文件
- E6：总标志文件存在且不允许覆盖 → 报错结束，结果为'E'

### 7.6.4 错误处理策略

| 场景 | 处理策略 |
|------|----------|
| 单文件生成失败 | 直接失败终止 |
| 多文件生成时发生错误 | 终止后续生成 |

### 7.6.5 空数据处理

当数据库无数据（记录数为0）时，根据配置决定行为：
- ERROR：结果为'E'，任务失败
- SEND_EMPTY：正常生成空文件，结果为'Y'
- SKIP：不生成文件，结果为'N'

## UC-07：轮询分桶明细

### 7.7.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-07 |
| 用例名称 | 轮询分桶明细 |
| 参与者 | FMSY系统节点 |
| 触发条件 | 节点竞争获得S类型子指令后 |
| 前置条件 | 存在分桶明细记录 |

### 7.7.2 事件流

**基本事件流**：
1. 从S类型子指令额外信息获取主指令ID
2. 循环执行以下步骤，直到没有可处理的S类型子指令：
   a. 尝试竞争一个S类型子指令：
      ```sql
      UPDATE 指令表
      SET 处理节点='节点ID', 处理状态='P'
      WHERE 类别代号='S' AND 处理状态='空' AND 处理节点 IS NULL
        AND 额外信息=主指令ID
      ```
   b. 如果竞争成功：
      - 获取该S类型子指令的额外信息（主指令ID）
      - 循环处理该主指令对应的就绪分桶：
        - 查询一批就绪分桶（一次获取多条）
        - 遍历这批分桶，逐条竞争处理：
          - 执行竞争UPDATE：
            ```sql
            UPDATE 明细表
            SET 处理节点='节点ID', 处理状态='P'
            WHERE 自增列=? AND 处理状态='空'
            ```
          - 如果影响行数=1，竞争成功，处理该分桶（分桶的稽核数由明细表记录）
          - 如果影响行数=0，跳过，处理下一条
        - 对每个竞争到的分桶：
          - 调用UC-11前稽核（使用明细表的稽核数，因为每个分桶数量不同）
          - 根据分桶的指定字段取值筛选数据
          - 生成文件到下发目录
          - 生成子文件标志文件（如 `file.dbf.ok`）
          - 调用UC-12后稽核
          - 更新分桶处理状态为'Y'（成功）或'E'（失败）或'N'（跳过）
        - 处理完这批后，继续轮询下一批分桶
      - 该S类型子指令处理完成，更新状态为'Y'或'E'
      - 调用UC-19写入该S类型子指令的结果记录
   c. 如果竞争失败，继续尝试竞争下一个S类型子指令

### 7.7.3 分桶处理状态流转

| 阶段 | 明细表状态 |
|------|------------|
| 分桶待处理 | 空（就绪） |
| 分桶正在处理 | P（处理中） |
| 分桶成功 | Y（成功） |
| 分桶跳过（空数据） | N（跳过） |
| 分桶失败 | E（异常） |

### 7.7.4 后置条件

- S类型子指令对应的分桶全部处理完成
- S类型子指令状态更新为'Y'或'E'
- 调用UC-19写入S类型子指令的结果记录

## UC-08：校验标志文件

### 7.8.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-08 |
| 用例名称 | 校验标志文件 |
| 参与者 | FMSY系统 |
| 触发条件 | 上传任务处理文件前 |
| 前置条件 | 配置了标志文件检测 |

### 7.8.2 事件流

**基本事件流**：
1. 解析传输配置中的标志文件配置（格式：`文件名模式;内容模式`）
2. 根据配置的标志文件名模式（如 `*.OK`, `*.FLG`）在FTP上查找
3. 如果未找到标志文件：
   - 结果为'N'，跳过文件处理，结束
4. 如果找到标志文件：
   a. 读取标志文件内容（期望值，如记录数）
   b. 检查内容是否匹配"空"模式（如NULL、EMPTY、0）：
      - 如果匹配"空"模式：跳过标志文件校验，结束
      - 如果不匹配：用数据文件内容计算实际校验值，与标志文件内容（期望值）比较
        - 一致：校验通过，结束
        - 不一致：标记异常，结果为'E'，结束
5. 返回校验结果

**校验逻辑说明**：
- 标志文件内容是外部系统写入的"期望值"（如记录数：`COUNT=1000`）
- 校验时用数据文件内容计算"实际值"，与期望值比较
- 与生成反馈文件相反：反馈文件是用数据文件计算值写入标志文件
- 常见校验内容：记录数、文件大小、校验码等

**内容模式说明**：
- 内容模式用于判断标志文件是否表示"无数据"
- 常见空模式值：NULL、EMPTY、0、空字符串
- 如果标志文件内容匹配空模式，表示数据文件可能无实际数据，跳过标志文件校验

**标志文件格式**：
- 常见后缀：OK、FLG、RET 等
- 内容格式：多种（如文本、JSON、INI等）
- 解析逻辑：固定写在代码中，根据配置选择解析方式

### 7.8.3 后置条件

- 标志文件不存在：文件被跳过，结果为'N'
- 标志文件内容匹配空模式：用数据文件内容校验，继续执行
- 校验通过：继续执行文件处理，结果可能为'Y'
- 校验失败：标记异常，结果为'E'

## UC-10：解析占位符

### 7.10.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-10 |
| 用例名称 | 解析占位符 |
| 参与者 | FMSY系统 |
| 触发条件 | 文件路径/文件名包含 `$变量名$` 格式 |
| 前置条件 | 路径配置中包含占位符 |

### 7.10.2 事件流

**基本事件流**：
1. 接收原始路径字符串
2. 识别所有 `$变量名$` 格式的占位符
3. 对每个占位符进行解析：
   a. **日期占位符**：如 `$YYYYMMDD$`、`$YYYYMMDDHHmmss$`
      - 使用当前日期/时间进行替换
   b. **数据库字段占位符**：如 `$TABLE_ID$`
      - 从指令表或配置表的对应字段取值
   c. **明细表占位符**：如 `$分桶值$`
      - 从明细表的指定字段取值获取
   d. **指令额外信息占位符**：如 `$EXTRA_INFO$`
      - 从指令表的额外信息字段取值
4. 执行替换，得到实际路径
5. 如果占位符无法解析，记录警告，使用默认值或跳过

## UC-11：执行前稽核

### 7.11.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-11 |
| 用例名称 | 执行前稽核 |
| 参与者 | FMSY系统 |
| 触发条件 | 文件处理前（下发前/上传前） |
| 前置条件 | 配置了稽核数或需要校验记录数 |

### 7.11.2 事件流

**上传场景（前稽核：比对稽核数和文件记录数）**：
1. 从指令表/明细表获取稽核数字段值
2. 如果稽核数 < 0，跳过前稽核，直接返回
3. 解析FTP上的文件，统计文件记录数（行数）
4. 比对：文件记录数 == 稽核数？
5. 如果相等，前稽核通过
6. 如果不相等，前稽核失败，返回失败

**下发场景（前稽核：比对稽核数和筛选后的数据库表记录数）**：
1. 从指令表/明细表获取稽核数字段值
2. 如果稽核数 < 0，跳过前稽核，直接返回
3. 执行SQL统计筛选后的目标表记录数
4. 比对：表记录数 == 稽核数？
5. 如果相等，前稽核通过
6. 如果不相等，前稽核失败，返回失败

### 7.11.3 后置条件

- 稽核通过：继续执行传输任务
- 稽核失败：返回失败，触发回滚

2. 执行SQL统计源表的记录数
3. 比对：文件记录数 == 表记录数？
4. 如果相等，后稽核通过
5. 如果不相等，后稽核失败，记录错误

### 7.12.3 后置条件

- 稽核通过：任务完成，结果为'Y'

## UC-17：检查串行约束

### 7.16.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-17 |
| 用例名称 | 检查串行约束 |
| 参与者 | FMSY系统节点 |
| 触发条件 | UC-01获取到指令后，竞争前 |
| 前置条件 | 已将正在处理的指令加载到内存Map |

### 7.16.2 事件流

**基本事件流**：
1. 获取指令的类别代号和控制代号
2. 如果是S类型指令（分桶子指令），从内存Map检查该节点是否已有正在执行的S类型指令：
   - 内存Map key: `处理节点+指令类型='S'` → 存在则跳过
   - 不存在则继续检查串行约束
3. 从内存传输配置中查询该类别代号和控制代号的串行标识
4. 如果串行标识='Y'：
   a. 从内存Map检查是否有任何节点在执行同代号指令：
      - key: `类别代号+控制代号` → 存在其他节点则跳过
      - 不存在则继续UC-02竞争
5. 如果串行标识='N'：
   a. 从内存Map检查该节点是否已有同代号指令在执行：
      - key: `类别代号+控制代号+当前节点ID` → 存在则跳过
      - 不存在则继续UC-02竞争

**特殊情况处理**：
- S类型子指令与主指令由相同节点处理时，不受串行约束限制
- 即：主节点在处理主指令期间，可以竞争处理该主指令的S类型子指令

**内存Map结构**：
```
Map< String, ProcessingInfo >
key = 类别代号 + 控制代号
value = { 节点ID列表 }

key = 类别代号 + 控制代号 + 节点ID
value = { 主指令ID, 是否有S类型子指令处理中 }
```

### 7.16.3 后置条件

- 串行约束检查通过：继续竞争指令
- 以下情况跳过该指令：
  - S类型指令：节点已有S类型指令在执行（排除同主指令情况）
  - 串行='Y'：有其他节点在执行同代号指令
  - 串行='N'：节点自身已有同代号指令在执行（排除同主指令的S类型子指令）

## UC-21：监控子指令完成

### 7.21.1 基本信息

| 属性 | 内容 |
|------|------|
| 用例ID | UC-21 |
| 用例名称 | 监控子指令完成 |
| 参与者 | FMSY系统（主节点） |
| 触发条件 | 主节点创建S类型子指令后 |
| 前置条件 | 存在待监控的S类型子指令 |

### 7.21.2 事件流

**基本事件流**：
1. 主节点在主指令的额外信息字段记录子指令数量
2. 主节点竞争获得一个S类型子指令后开始处理分桶
3. 每次处理完一个分桶，更新该S类型子指令的状态为'Y'/'E'
4. S类型子指令完成后，主节点减少剩余计数
5. 主节点继续轮询处理下一个可竞争的S类型子指令
6. 当主节点处理的S类型子指令完成（状态='Y'或'E'）时：
   a. 检查是否还有其他S类型子指令正在处理
   b. 如果有，继续等待
   c. 如果没有（所有S类型子指令都完成），汇总结果
7. 汇总所有分桶的处理结果
8. 更新主指令状态为'Y'（全部成功）或'E'（部分失败）或'N'（全部跳过）
9. 调用UC-19写入主指令的结果记录
10. 所有分桶文件处理完成后，根据主指令状态决定是否覆盖总标志文件：
    - 如果主指令状态为'Y'，覆盖生成总标志文件（表示任务完成）
    - 如果主指令状态为'N'，无数据文件，无需生成总标志文件
    - 如果主指令状态为'E'，不覆盖总标志文件（表示任务异常）

**计数方式**：
- 主节点在插入S类型子指令时，将数量记录在主指令的额外信息字段
- 每完成一个S类型子指令，计数减1
- 计数归0时表示所有子指令完成

### 7.21.3 后置条件

- 所有S类型子指令完成
- 主指令状态更新为'Y'或'E'
- 总标志文件已覆盖（如全部成功）

## 9. 转换器规格

### 9.1 设计原则

**流式处理原则**：
- 上传场景：边读文件边解析，分批次写入数据库，避免大文件超过内存
- 下发场景：边读数据库边生成文件，分批次写入文件，避免大查询结果超过内存

**插件式架构**：
- 每种文件格式独立转换器
- 实现统一接口
- 转换器内嵌默认参数，配置表只覆盖需要自定义的部分
- 新增格式只需实现转换器接口，配置表配置即可启用

### 9.2 转换器接口定义

```java
public interface FileConverter {
    /**
     * 边读文件边解析，分批次返回数据（流式处理）
     * @param input 文件输入流
     * @param mapping 字段映射
     * @return 分批次的数据迭代器，每批次达到一定量后即返回
     */
    Iterator<List<Map<String, Object>>> parse(InputStream input, FieldMapping mapping);

    /**
     * 分批次生成文件（流式写入）
     * @param output 文件输出流
     * @param data 分批次的数据迭代器
     * @param mapping 字段映射
     */
    void generate(OutputStream output, Iterator<List<Map<String, Object>>> data, FieldMapping mapping);

    // 获取支持的格式
    String getFormat();

    // 获取默认配置
    Map<String, String> getDefaultConfig();
}
```

### 9.2.1 字段映射机制

#### 上传字段映射（文件→数据库）

1. 读取目标数据库表的元数据，获取字段列表（按表定义顺序）
2. 读取配置表的忽略字段配置（忽略字段列表），从元数据字段列表中去除，得到最终字段列表
3. 如为明细表指定上传（指定字段取值非空），从最终字段列表中去除指定字段名称
4. 读取数据文件，按文件格式解析出字段顺序
5. 将数据文件的字段值与最终字段列表一一对应（按顺序）
6. 补充指定字段键值（从明细表的指定字段取值）

**示例**：数据库表字段：A, B, C, D；忽略字段配置：C；明细表指定字段名称：B,E；指定字段取值：val1,val2
```
Step1：数据库表字段：A, B, C, D
Step2：去除忽略字段C → A, B, D
Step3：去除指定字段B,E → A, D（此时只有2个字段，数据文件也只有2个值）
Step4：数据文件读值：[vA, vC]
Step5：对齐后：A=vA, D=vC
Step6：补充指定字段：A=vA, B=val1, D=vC, E=val2
```

#### 下发字段映射（数据库→文件）

1. 读取源数据库表的元数据，获取字段列表（按表定义顺序）
2. 读取配置表的忽略字段配置（忽略字段列表），从元数据字段列表中去除，得到最终字段列表
3. 从数据库读取数据记录
4. 按最终字段列表顺序生成文件

**说明**：下发时忽略字段由配置表忽略字段配置指定，与明细表无关。

### 9.3 DBFConverter

**能力**：
- 解析：读取DBF文件，返回记录列表
- 生成：创建DBF文件，写入记录
- 字段映射：DBF字段名直接对应文件Header

**默认参数**：
- 字段名称最大长度：10字符
- 字段类型支持：字符（C）、数字（N）、日期（D）、逻辑（L）

#### 解析事件流（DBF → 数据库）

1. 读取DBF文件头（3字节版本标识 + 修正日期 + 记录数 + 头长度）
2. 解析字段描述区（字段名12字节 + 类型1字节 + 长度1字节 + 小数位1字节）
3. 提取字段定义列表：字段名、类型、长度、小数位
4. 读取数据记录区（每条记录以删除标志开头，数据按字段长度定长读取）
5. 按字段定义解析每条记录的各个字段值
6. 转换为Map列表返回（key为字段名，value为字段值）

#### 生成事件流（数据库 → DBF）

1. 创建DBF文件头（版本标识、修正日期、记录数、头长度、记录长度）
2. 按字段定义写入字段描述区
3. 写入头结束标记（0x0D）
4. 对每条数据记录：
   - 写入删除标志（空格）
   - 按字段定义写入各字段值（定长，不足补空格/零，超长截断）
5. 写入文件结束标记（0x1A）

**解析配置参数**（可选）：
- `encoding`：文件编码，默认GBK

### 9.4 XMLConverter

**能力**：
- 解析：读取XML文件，按标签/属性提取数据
- 生成：创建XML文件，按结构写入数据

**默认参数**：
- 标签映射：按配置或自动识别
- 属性提取：按配置

#### 解析事件流（XML → 数据库）

1. 读取XML文件内容
2. 解析XML声明（获取编码）
3. 解析根元素开始标签
4. 遍历子元素（行/记录）：
   - 解析元素标签获取字段名
   - 解析元素内容获取字段值
   - 支持嵌套结构（多级标签映射到展平字段）
5. 构建Map列表返回

#### 生成事件流（数据库 → XML）

1. 创建XML文档，写入XML声明（版本1.0，编码）
2. 写入根元素开始标签
3. 对每条数据记录：
   - 写入记录元素标签
   - 对每个字段：写入字段元素标签 + 内容 + 结束标签
4. 写入根元素结束标签

**解析配置参数**（可选）：
- `encoding`：文件编码，默认UTF-8
- `rootElement`：根元素名称，默认`data`
- `recordElement`：记录元素名称，默认`record`
- `fieldAsAttribute`：是否将字段作为属性输出，默认false

### 9.5 CSVConverter

**能力**：
- 解析：读取CSV文件，按分隔符解析
- 生成：创建CSV文件，按分隔符写入

**默认参数**：
- 分隔符：逗号
- 引用符：双引号
- 编码：UTF-8

#### 解析事件流（CSV → 数据库）

1. 读取CSV文件第一行（表头行）
2. 解析表头字段名列表
3. 读取数据行（从第二行开始）
4. 对每行数据：
   - 按分隔符分割行
   - 处理引用符包围的字段（引号内部分隔符不分割）
   - 去除前后空白
   - 按表头字段顺序构建Map
5. 返回Map列表

#### 生成事件流（数据库 → CSV）

1. 创建CSV文件，写入表头行（字段名用分隔符连接）
2. 对每条数据记录：
   - 将字段值转换为字符串
   - 需要引号的字段（包含分隔符/换行符/引号）用引号包围
   - 用分隔符连接各字段值
   - 写入一行
3. 关闭文件

**解析配置参数**（可选）：
- `encoding`：文件编码，默认UTF-8
- `separator`：分隔符，默认逗号
- `quote`：引用符，默认双引号
- `skipLines`：跳过行数（表头行），默认1
- `headers`：指定表头字段列表（可覆盖文件内表头）

### 9.6 TXTConverter

**能力**：
- 解析：读取纯文本文件，按固定宽度或分隔符解析
- 生成：创建纯文本文件

**默认参数**：
- 分隔符：制表符（可配置）
- 或字段宽度配置（定长）

#### 解析事件流（TXT → 数据库）

**模式A：分隔符模式**
1. 读取TXT文件第一行（表头行）
2. 按分隔符解析表头字段名列表
3. 读取数据行
4. 按分隔符分割，映射到表头字段
5. 返回Map列表

**模式B：定长模式**
1. 读取字段宽度配置
2. 读取数据行
3. 按宽度截取各字段值
4. 映射到字段名
5. 返回Map列表

#### 生成事件流（数据库 → TXT）

**模式A：分隔符模式**
1. 创建TXT文件，写入表头行（字段用分隔符连接）
2. 对每条数据记录：
   - 各字段值转换为字符串
   - 用分隔符连接
   - 写入一行

**模式B：定长模式**
1. 获取字段宽度配置
2. 对每条数据记录：
   - 各字段值按宽度截断或补齐
   - 拼接成一行
   - 写入文件

**解析配置参数**（可选）：
- `encoding`：文件编码，默认UTF-8
- `mode`：模式，分隔符（delimiter）或定长（fixed）
- `separator`：分隔符（delimiter模式），默认制表符
- `fieldWidths`：字段宽度列表（fixed模式），如`10,20,15`
- `skipLines`：跳过行数，默认0

### 10.1.3 前置文件操作详解

**格式**：`操作类型;参数,操作类型;参数,...`

| 操作类型 | 参数 | 说明 |
|----------|------|------|
| CHECK_READY | 文件名模式 | 检查文件是否存在，不存在则跳过（结果为'N'），结束 |
| CHECK_FLAG | 文件名模式;内容模式 | 检查标志文件是否存在，不存在则跳过；存在则读取内容（期望值），若匹配"空"模式则跳过校验，否则用数据文件内容计算实际值与期望值比较 |

**校验逻辑说明**：
- 标志文件内容是外部系统写入的"期望值"（如记录数：`COUNT=1000`）
- 校验时用数据文件内容计算"实际值"，与期望值比较
- 与生成反馈文件相反：反馈文件是用数据文件计算值写入标志文件

**内容模式说明**：
- 标志文件内容匹配内容模式（如NULL、EMPTY、0）时，表示无实际数据，跳过校验

**配置示例**：
- `*.OK` - 检查*.OK文件，不存在则跳过
- `*.OK;NULL,*.FLG` - 检查*.OK，不存在则N；*.FLG存在时若内容为NULL则用数据文件校验
- `*.OK,*.FLG;0` - 检查*.OK；检查*.FLG，若内容为0则用数据文件校验

## 11. 非功能性需求

### 11.1 性能需求

**轮询与指令获取**：
- 轮询间隔：5-300秒可配置，建议值5-10秒（高吞吐场景可设5秒）
- 每批次获取指令数：默认20，可配置，建议值50-100（高吞吐场景）
- 竞争策略：每批次获取后逐条竞争，每批次指令数越大并发处理越多

**并发处理**：
- 单节点并发处理线程数：默认10，可配置（配置于系统参数）
- 单节点多线程上传并发数：默认3，避免事务爆炸
- 分桶明细每批获取数：默认3
- 单文件处理超时：可配置，默认1小时（UC-01运行时检测）

**多节点扩展**：
- 最大并发节点数：无限制，由指令数量决定
- 节点间竞争：通过数据库UPDATE竞争，无锁竞争
- 吞吐预估：单节点10并发 × 50批次/轮询 × N节点 = 可线性扩展

**高吞吐配置建议**：
| 场景 | 轮询间隔 | 每批次获取 | 单节点并发 |
|------|----------|-----------|------------|
| 默认 | 10秒 | 20 | 10 |
| 高吞吐 | 5秒 | 50-100 | 10-20 |

### 11.2 可用性需求

- 多活节点部署，自动故障转移
- 节点故障时，其持有的指令自动释放：
  - 运行时：UC-01轮询检测超时任务（1小时），超时自动释放为'E'
  - 启动时：UC-22重置上次中断的P状态指令为'N'
- 任务失败重试次数可配置

### 11.2.1 FTP主备切换支持

**背景**：当FTP服务器配置为主备集群时（通过DNS轮询或/etc/hosts实现同一主机名映射多个IP），应用需支持主备自动切换。

**配置方式**：
```yaml
ftp:
  config:
    - id: FTP_DEFAULT
      host: ftp.example.com    # /etc/hosts中配置主备IP映射
      port: 21
      username: ftp_user
      password: secret
      failover:
        enabled: true          # 启用主备切换
        maxRetries: 2           # 最大重试次数
      healthCheck:
        enabled: true           # 启用连接健康检查
        intervalSeconds: 30     # 健康检查间隔
```

**/etc/hosts配置示例**：
```
192.168.1.101  ftp.example.com  # 主
192.168.1.102  ftp.example.com  # 备
```

**故障切换流程**：
1. 应用通过`InetAddress.getAllByName()`解析主机名获取所有IP
2. 按顺序尝试每个IP进行连接
3. 连接成功则返回，失败则尝试下一个
4. 所有IP尝试失败后抛出异常

**连接池保护机制**：
| 时机 | 保护机制 | 说明 |
|------|---------|------|
| 获取连接时 | sendNoOp()验证 | 无效连接不返回 |
| 归还连接时 | 有效性检查 | 无效连接从池中移除 |
| 健康检查时 | 定时检测 | 失效连接自动清理 |
| 操作失败时 | 标记移除 | 失败后标记连接为dead |

**主备切换保证**：从连接池获取的连接都经过有效性验证，不会是已失效的旧节点连接。

### 11.3 安全需求

- 数据库密码、FTP密码存储在配置文件中，不明文出现在代码
- 支持不同环境使用不同配置文件

### 11.4 可扩展性需求

- 新增文件格式只需实现转换器接口，配置表配置即可启用
- 无需修改公共流程代码
- 转换器内嵌默认参数，格式参数解耦

### 11.5 日志需求

- 本地文本日志，分级记录
- 日志文件按日期滚动
- 外部程序可采集日志做监控
