# 单文件上传（UPLOAD_SINGLE）单元测试覆盖报告

> 生成日期: 2026-07-12
> 测试总数: 1011 (全量), 其中单文件上传相关: **36 用例**

---

## 一、测试架构

```
SingleUploadHandler.handle(command, config, result)
    │
    ├── Phase 1: preCheck ─────────────────────────→ TransferSupport.preCheck()
    │                                                    └── FlagFileService.preCheck()
    │
    ├── Phase 2: preAudit ─────────────────────────→ UploadSupport.preAudit()
    │
    ├── Phase 2: readAndInsert ───────────────────→ UploadSupport.insertAndVerifyInTx()
    │   ├── FTP stream open & file parse
    │   ├── transaction begin
    │   │   ├── truncate (if clearTableFlag=Y)
    │   │   ├── batch insert loop
    │   │   ├── handleEmptyData check
    │   │   └── postAudit check (rollback if ERROR)
    │   └── transaction commit / rollback
    │
    └── Phase 3: postProcess ───────────────────→ TransferSupport.postProcess()
                                                        └── FlagFileService.process()
```

## 二、SingleUploadHandler 测试覆盖

**测试类**: `SingleUploadHandlerTest`  
**测试框架**: Mockito + JUnit 5  
**Mock 策略**: UploadSupport / TransferSupport / ConverterFactory / FieldMappingBuilder 全部 Mock，仅测试 Handler 编排逻辑

### 2.1 已覆盖场景（14 用例）

| # | 场景 | 测试类 | 触发 | 验证点 | 用例数 |
|---|------|-------|------|-------|:-----:|
| H1 | **PreCheck 失败** | `HandlePreCheckFailure` | `preCheck()` → false | STATUS_SKIPPED, recordCount=0, 不调 converterFactory/uploadSupport | 1 |
| H2 | **PreAudit 失败** | `HandlePreAuditFailure` | `preAudit()` → -1 | STATUS_ERROR, file moved to error dir | 1 |
| H3 | **完整成功流程** | `HandleSuccessfulFlow` | 全流程通过 | STATUS_SUCCESS, recordCount=100, postProcess 被调用 | 2 |
| H4 | **后审计失败（非ERROR）** | `HandlePostAuditFailure` | postAudit mock→false, non-ERROR | STATUS_ERROR, file moved to error dir | 1 |
| H5 | **Phase 3 postProcess 异常** | `HandlePhase3Exception` | postProcess 抛出异常 | STATUS_ERROR, description 含 "uploaded successfully (100 records)" | 1 |
| H6 | **FlagCheckException 传播** | `HandlePreCheckFlagCheckException` | preCheck 抛出 FlagCheckException | 异常传播到 Orchestrator→STATUS_ERROR | 1 |
| H7 | **空文件 + ALLOW 模式** | `HandleEmptyFile` | 0 records, ALLOW | STATUS_SUCCESS, recordCount=0, postProcess(0) 被调用 | 1 |
| H8 | **空文件 + ERROR 模式** | `HandleEmptyFile` | 0 records, ERROR | Runtime 异常传播到 Orchestrator | 1 |
| H9 | **moveToErrorDir 异常安全** | `HandleMoveToErrorDirException` | moveToErrorDir 抛出异常 | 异常被 catch, STATUS_ERROR 正常设置 | 1 |
| H10 | **truncateFirst 参数传递** | `HandleTruncateFirst` | clearTableFlag=Y | `insertAndVerifyInTx` 收到 truncateFirst=true | 1 |

### 2.2 分支覆盖矩阵

```
SingleUploadHandler.handle()
├── resolveFilePath ─────────────────────────────────── 所有测试间接覆盖
│
├── executeWithClient + preCheck ───── H1 (false) ──── STATUS_SKIPPED
│                                       H6 (异常) ──── 异常→Orchestrator→ERROR
│
├── converterFactory.get ──────────── H3 (CSV 正常)
│
├── preAudit ──────────────────────── H2 (-1) ──────── STATUS_ERROR + moveToErrorDir
│                                     H3 (0) ───────── 继续
│
├── readAndInsert ─────────────────── H3 (count>=0) ── Phase 3
│                                     H4 (result→ERROR) ── STATUS_ERROR + moveToErrorDir
│                                     H7 (0, ALLOW) ── Phase 3
│                                     H8 (异常) ─────── 异常→Orchestrator
│
└── postProcess ───────────────────── H3 (正常) ────── STATUS_SUCCESS
                                      H5 (异常) ────── STATUS_ERROR + accurate desc
```

### 2.3 Handler 层覆盖率

| 分支 | 覆盖情况 |
|------|:--------:|
| `preCheckOk == false` | ✅ H1 |

| `recordCount >= 0` → postProcess 成功 | ✅ H3 |
| `recordCount >= 0` → postProcess 异常 | ✅ H5 |
| `recordCount < 0`（readAndInsert 返回-1） | ✅ H4 |
| preCheck 抛出 FlagCheckException | ✅ H6 |

## 五、场景流程图（分支 + 状态）

```
handle()
  │
  ├─ resolveFilePath ───────────────────────────────────
  │                                                    │
  ├─ executeWithClient(ftpName, preCheck) ──────────────┤
  │   ├─ ✅ preCheck 返回 true ──────────────────────── 继续
  │   ├─ ❌ preCheck 返回 false ─────────────────────── STATUS_SKIPPED
  │   └─ ⚠ preCheck 抛出 FlagCheckException ────────── 异常→Orchestrator→STATUS_ERROR
  │                                                    │
  ├─ converterFactory.get(parserType) ──────────────────┤
  │   ├─ ✅ 支持格式 ─────────────────────────────────  继续
  │   └─ ❌ 不支持格式 ──────────────────────────────── IllegalArgumentException→Orchestrator
  │                                                    │
  ├─ support.preAudit() ────────────────────────────────┤
  │   ├─ ✅ 返回 0（通过/无需审计）──────────────────── 继续
  │   ├─ ❌ 返回 -1（不通过）────────────────────────── STATUS_ERROR + moveToErrorDir
  │   └─ ⚠ countRecords=-1（XML）───────────────────── 返回 0, 跳过
  │                                                    │
  ├─ readAndInsert() ───────────────────────────────────┤
  │   │
  │   ├─ executeWithClient(ftpName, callback) ──────────┤
  │   │   │                                             │
  │   │   ├─ getInputStream(filePath) ──────────────────┤
  │   │   │                                             │
  │   │   ├─ buildForUpload(config) → FieldMapping ─────┤
  │   │   │                                             │
  │   │   ├─ converter.parse(is, mapping) ──────────────┤
  │   │   │   → CloseableIterator                       │
  │   │   │                                             │
  │   │   └─ support.insertAndVerifyInTx() ─────────────┤
  │   │       │  [transaction begin]                    │
  │   │       ├─ truncate (if clearTableFlag=Y) ────────┤
  │   │       ├─ batch insert loop                     │
  │   │       ├─ handleEmptyData(totalRecords) ─────────┤
  │   │       │   ├─ ALLOW → continue                  │
  │   │       │   ├─ ERROR → 抛出异常 → rollback      │
  │   │       │   └─ SKIP → 抛出异常 → rollback       │
  │   │       ├─ postAudit(fileRecords, inserted) ──────┤
  │   │       │   ├─ 通过 → commit → return count      │
  │   │       │   ├─ 不通过+ERROR → 抛出异常→rollback  │
  │   │       │   └─ 不通过+非ERROR → commit,          │
  │   │       │        result.setOutcome(0, ERROR)      │
  │   │       └─ [transaction commit/rollback]          │
  │   │                                                │
  │   ├─ result 为 ERROR → moveToErrorDir → return -1  │
  │   └─ result 正常 → return count                     │
  │                                                    │
  ├─ recordCount >= 0 ? ────────────────────────────────┤
  │   ├─ ❌ (result 已设置 ERROR) return                │
  │   └─ ✅ Phase 3: postProcess ──────────────────────┤
  │       ├─ 成功 → STATUS_SUCCESS                      │
  │       └─ 异常 → STATUS_ERROR + 记录数 + 异常详情    │
  │
  └─ Orchestrator 兜底: 任何未捕获异常 → failWith(e) → STATUS_ERROR
```
